本仓库是一个**硕士毕业论文项目**（非传统软件工程项目），包含 MATLAB 仿真代码、Markdown 论文章节以及 AI 辅助写作/评审工作流系统。

**论文题目：**《基于LFMCW时延轨迹特征与参数反演的等离子体诊断及色散等效研究》

**研究领域：** LFMCW 雷达信号处理、等离子体诊断、贝叶斯反演（MCMC）

## 构建 / 运行 / 测试命令

本项目无传统构建系统，使用 MATLAB 和 Markdown 进行开发与写作。

## 不可违背的理论约束

以下为已确立的物理结论，在任何论文文本或代码中**绝对不可违背**：

1. **碰撞频率是二阶微扰**：$\nu_e$ 对群时延的贡献量级为 $(\nu_e/\omega)^2$（二阶小量），而对幅度衰减的贡献是一阶主导。
2. **反演策略**：将 $\nu_e$ 固定为常数，仅反演 $n_e$。**绝不可**建议同时反演两者。
3. **非线性度因子**（分母必须是 1.5 次方）：
   $\eta(f) = \frac{B}{f} \cdot \frac{(f_p/f)^2}{[1-(f_p/f)^2]^{1.5}}$
4. **工程适用性判据（局部）**：传统全频段 FFT 单峰方法的局部工程适用边界应写为 $\chi(f_c)=B \cdot \eta(f_c) \cdot \tau_0 \lesssim 1$，其中 $f_c$ 为代表频率。该判据仅在工作带宽完全位于可传播区且局部线性化余项受控时成立；若不满足，则应回到全频段模型或数值仿真评估，**不得**将其写成全局无条件判据。
5. **技术路线**：滑动窗口 → MDL 信源估计 → ESPRIT 超分辨 → MCMC 贝叶斯反演
6. **第五章的定位不可写偏**：第五章引入微波带通滤波器的目的**不是**诊断或识别滤波器物理参数本身，而是将其作为“色散函数数学形式已知、真值可控”的等效验证载体，用于验证本文算法能否从 LFMCW 差频信号中稳定提取色散介质的时延特征点，并证明只要色散正向模型的数学形式已知，这些特征点即可进一步驱动后续参数反演计算。后续写作中**禁止**把第五章表述为“滤波器诊断”或“以识别滤波器参数为研究目标”。

## MATLAB 代码规范

### 文件头部

每个脚本以装饰性注释块和初始化语句开头：
```matlab
%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
% 脚本功能描述（中文）
% 用途：...
%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
clc; clear; close all;
```

### 命名约定

| 类别 | 规则 | 示例 |
|------|------|------|
| 变量 | snake_case，结合物理符号 | `f_start`, `omega_p`, `n_e`, `tau_air` |
| 信号变量 | 时域用 `s_` 前缀，频域用 `S_` 前缀 | `s_tx`, `S_IF_air` |
| 函数 | snake_case | `calculate_theoretical_delay()`, `compute_log_likelihood()` |
| 优化器代价函数 | PascalCase（唯一例外） | `WeightedResiduals_Scaled()` |
| 特征数组 | `feature_` 前缀 | `feature_f_probe`, `feature_tau_absolute` |
| 算法脚本文件名 | PascalCase + 下划线 | `LM_MCMC.m`, `LM_lorentz.m` |
| 绘图脚本文件名 | `plot_fig_{章}_{节}_{描述}.m` | `plot_fig_4_1_sensitivity_comparison.m` |

### 文件内部结构

脚本按以下顺序组织：
1. 文件头注释块
2. `clc; clear; close all;`
3. 物理常数与参数设置
4. 信号生成
5. 传播仿真（频域处理）
6. 混频 / 差频信号提取
7. 高级信号处理（ESPRIT/MDL）—— `%% 7. ...` 节
8. 参数反演（LM 或 MCMC）—— `%% 9. ...` 节
9. 结果输出（`fprintf`）
10. 局部函数置于文件最末尾

### 错误处理

- **try/catch 配合哨兵值**用于优化回调函数中：
  - 失败时返回 `ones(size(f_data)) * 1e5` 作为大残差
  - 失败时返回 `-1e10` 作为极负对数似然
- **物理约束前置检查**（计算前提前返回）：
  ```matlab
  if ne_val <= 0, logL = -1e10; return; end
  ```
- **NaN/Inf 防护**在每次计算后执行：
  ```matlab
  if isnan(logL) || isinf(logL), logL = -1e10; end
  ```
- `error()` 仅用于不可恢复的致命数据问题。

### 注释规范

- 节标题：`%% N. 标题`（编号的单元分隔符）
- 小节分隔符：`% --- 描述 ---`
- 双语注释：中文用于物理解释，英文用于公式和 MATLAB 技术说明
- 进度日志：`fprintf('【步骤N】描述...\n');`

### 其他约定

- 数组预分配：循环前使用 `zeros(N_samples, 1)`
- 随机种子：随机采样前设置 `rng(42)` 确保可复现
- 频域安全处理：将 `omega == 0` 替换为 `1e-10` 避免除零
- 续行符：使用 `...`，后续行对齐缩进
- 物理常数在每个文件顶部重新定义（无共享常数文件）
- 长时间循环（ESPRIT/MCMC）使用 `waitbar` 显示进度
- 图形设置：`figure(N); clf; set(gcf, 'Position', [...]);`，字体使用 `'FontName', 'SimHei'`

## 文件编码与防乱码规范（新增）

为避免中文乱码，后续所有智能体与人工修改必须遵循以下规则：

1. **编码基准**
   - `.m`、`.md`、`.txt` 默认使用 `UTF-8`（无 BOM）。
   - 未经明确要求，不得单独改某个文件编码为 GBK/ANSI。

2. **写入方式限制**
   - 修改中文文件时，优先使用补丁式编辑（如 `apply_patch`）。
   - 禁止使用可能改变编码的整文件重写方式（如 PowerShell `Set-Content` 未明确编码）。
   - 若必须脚本化写回，必须显式指定编码，并与原文件保持一致。

3. **修改后强制自检**
   - 检查是否存在明显乱码特征：`锟�`、`��`、`銆�`、`娣烽` 等异常串。
   - 检查字符串是否闭合，避免因乱码导致引号丢失引发 MATLAB 语法错误。
   - 检查同目录同类型文件编码一致性，禁止“单文件异编码”。

4. **跨工具协作要求**
   - 若 MATLAB 显示正常但终端显示乱码，以 MATLAB 打开效果为准，并记录实际编码。

5. **故障应急流程**
   - 先备份当前文件。
   - 再做统一转码 + 文本修复。
   - 最后最小化改动恢复功能，确保脚本可运行。

## 论文写作规范

### 语言规则

- **禁止**使用第一人称（"我"、"我们"）
- **必须**使用"本文"、"本系统"、"该方法"、"实验表明"
- 因果状语前置："为了解决...，本文提出了..."
- 每章每节末尾必须有总结性段落
- 正文章节的子标题数量必须克制；除章节、节、必要的小节层级外，不得为制造结构感而频繁切分出过密的三级、四级标题，避免正文呈现明显的 AI 生成痕迹或讲义式碎片化版式
- 正文论述应以连续、完整的学术段落为主，不能依赖大量并列小标题、提示语或机械分块来推动叙述；若删除某个子标题后段落仍能自然衔接，则该子标题通常不应保留
- 正文单段长度与单节篇幅都应适度控制，符合硕士论文的常规阅读节奏；禁止将单一论点拖成长段重复铺陈，也禁止为追求篇幅而堆砌背景、同义改写和显然结论
- 写作时必须主动检查：**逻辑密度是否足够、AI腔是否明显、是否存在重复铺陈、术语口径是否前后一致（理论、符号统一、公式编号、篇幅控制和语言质感）**
- 语言必须精练，句子应直接承担定义、推导、比较、解释或结论功能；不能出现大段不承载新信息的过渡话、套话、评价话和空泛总结
- 每段文字都应有明确的信息增量；若相邻两段只是换一种说法重复同一主张，应合并压缩，避免“扩写型”写法
- 禁止使用“空泛总结 + 套话连接 + 方法名堆叠”的AI式表述，如“围绕上述问题，本文研究了……”“具有重要理论意义和工程价值”等泛化句式连续出现
- 禁止使用带引号的“命名式总结句”或“口号式范式表达”来硬性包装观点，如“提出了从‘色散消除’到‘色散利用’的范式转换”“关键矛盾并不在于‘能否产生差频信号’”。应直接陈述物理矛盾、方法变化与因果关系，不用引号制造概念感
- 同一核心主张在相邻段落中不得反复换句重说；若前文已明确提出“色散利用”“群时延轨迹”“降维反演”等核心观点，后文应在新信息基础上推进，而非仅作同义重复
- 术语必须全书统一，尤其要保持“群时延轨迹”“参数反演”“降维策略”“色散等效验证载体”等表述稳定，禁止同一概念在不同章节随意改写为不同说法
- 绪论、摘要和章节小结应避免“提纲展开体”或“答题卡体”写法，不得写成明显的AI生成结构
- 正文写作的时候不能引用 `thesis-code/LM_MCMC_with_noise.m`文件这样的类似表达

### 公式与术语

- 所有公式使用行内 LaTeX 格式：`$公式$`
- 公式格式参照 `resources/formula-templates.md`
- 核心术语（必须统一使用）：LFMCW、差频信号、群时延、栅栏效应、信噪比、MDL 准则、ESPRIT、Levenberg-Marquardt
- 正文章节中的公式必须遵循“必要、可回引、服务主线”的原则；仅保留支撑物理建模、误差分析、算法推导或后文讨论所必需的公式，教材式中间变形、一次性过渡公式和不再使用的展开式应压缩或删除
- 公式编号必须服务于正文引用；凡后文会再次引用、比较、变形或作为判据/模型基础的公式必须编号，未被正文回引的过渡公式原则上不编号
- 公式编号在全书内必须统一采用同一规则；默认按章连续编号，如 `(3-12)`、`(4-7)`，除非学校模板另有强制要求，否则不得在同一论文中混用“按章编号”“按节编号”“全书连续编号”三种方式
- 文中引用公式的写法必须统一，例如统一写为“由式(3-12)可知”，禁止“式(3-12)”“公式3.12”“Eq. (3-12)”等多套表述混用
- 全文字母、下标、角标和缩写必须统一；同一物理量只能保留一种主符号写法，同一符号不得在不同章节承担不同物理含义
- 首次出现的符号必须在正文中就地定义，说明其物理意义、单位、适用条件或参考状态；禁止在未定义的情况下直接引入 `\eta(f)`、`\tau_0`、`f_p`、`\omega_p`、`S` 等记号
- 需要全文重点统一的符号包括但不限于：`n_e`、`\nu_e`、`f_p`、`\omega_p`、`\tau_g`、`\tau_0`、`B`、`T`、`S=B/T`、`\eta(f)`；其中 `\nu_e` 的表述必须始终与“固定为常数、仅作为二阶微扰项、不参与联合反演”的理论约束一致
- 正文、摘要、章节小结、图注、表注和 MATLAB 代码中的关键符号口径必须一致；若代码变量名与正文记号不同，必须在文中或注释中建立明确映射，不得让同一物理量出现多套未说明的符号系统
- 每个编号公式都应自检三项：后文是否会引用、是否属于关键定义/关键判据/关键模型、删去后是否影响后续理解；若三项均不满足，则该式通常不应编号，必要时也不应保留
- 推导写作应有止损点：正文只推导到足以支撑后续分析的位置，过长的代数化简、标准积分过程或附加证明应转入附录、脚注或文字概述，避免公式堆叠削弱主线
