# DOCX Sync Workflow

## Scope

This skill is intentionally narrow. Use it for section-level thesis updates where the Markdown already contains the final wording and the master document is a `.docx` file with stable heading styles.

Supported in v3:

- heading-bounded section replacement
- chapter-level sync after merging multiple Markdown section files into one temporary chapter file
- front-matter anchors that use a non-numbered title style, such as `摘要` and `ABSTRACT`
- Markdown headings whose visible numbering should be stripped because the target Word heading style already numbers them
- body paragraphs
- ordered and unordered lists
- standard Word tables produced by `pandoc`
- inline and display math rendered by `pandoc`
- display equations that include `\tag{...}` in Markdown
- Markdown image references that can be resolved to local files inside the thesis workspace
- optional downgrade of unresolved Markdown images into visible placeholder text
- footnotes and endnotes generated by `pandoc`
- external hyperlinks generated by `pandoc`
- internal anchors, bookmarks, and in-document jump links generated from Markdown IDs or `#anchor` links
- table of contents, figure index, and table index refresh compatibility after Word field update

Still not supported:

- legacy `.doc`
- tracked changes
- Word comments
- embedded OLE objects inside the Markdown fragment
- Markdown fragments whose image paths cannot be resolved to real local files
- equation-number layout that must exactly mimic Word right-aligned professional equation templates

## Recommended Flow

1. Work on the chapter in Markdown until the content is ready.
2. If the chapter is split into multiple Markdown files, merge them first with `build_markdown_chapter.py`. Put the merged file near the source files so relative image paths such as `figures/...` still resolve.
3. Run `dump_docx_outline.py` on the target master file and copy the exact heading text you want to replace.
4. If the master file is an older thesis version, confirm the target block by surrounding headings and body position instead of assuming the chapter numbering still matches the Markdown numbering.
5. Run `sync_markdown_to_docx.py` with that heading as the anchor.
6. If the output file is open in Word, close it or write to a new output filename first.
7. Open the output `.docx` in Word and refresh table of contents, figure index, and table index if needed.
8. Do a fast visual check on formulas, tables, paragraph spacing, and chapter breaks.
9. If a formula used `\tag{...}`, confirm the number is still visually acceptable after conversion.
10. If the chapter contains footnotes or internal jump links, click a few representative items in Word to confirm they still resolve correctly.

## Why This Works

The sync script does not regenerate the whole thesis document. It only replaces the XML block between one heading and the next sibling heading of the same or higher level. This keeps the rest of the master file unchanged, including front matter, references, and other chapters.

`pandoc` is used only to render Markdown into a Word fragment that already follows the style universe of the thesis master document through `--reference-doc`.

Before rendering, the script rewrites display equations containing `\tag{...}` into a `pandoc`-compatible form and stages Markdown images into a temporary asset folder. During DOCX merge, it also carries over document relationships, bookmark IDs, and note parts so images, hyperlinks, footnotes, endnotes, and internal jump targets survive the section replacement.

When the target anchor uses a non-numbered title style instead of a numbered heading style, the script also aligns the first inserted heading paragraph to that target style so abstract-like sections keep the template's original appearance.

## Failure Modes

- If the heading cannot be found, dump the outline first and use the exact text from the target `.docx`.
- If the heading is ambiguous, rerun with `--level`.
- If the replacement lands in the wrong chapter, the likely cause is a stale master `.docx` whose chapter order no longer matches the Markdown plan. Re-check the actual Word outline and surrounding headings before rerunning.
- If the chapter is split across several Markdown files, merge them first instead of trying to sync each subsection into a chapter-level anchor manually.
- If the script reports unsupported features, move comments or OLE objects back to the master document manually and sync the rest.
- If the script reports unresolved image paths, fix the Markdown path or move the referenced image into the workspace first.
- If the source images are genuinely unavailable but the text sync still needs to continue, rerun with `--missing-images placeholder`.
- If the script reports the output DOCX is locked or open, close the file in Word or change `--output`.
- If the table of contents or figure index looks stale after sync, update fields in Word instead of editing those index pages manually.
- If the master file is still `.doc`, convert it to `.docx` first and treat the converted file as the new single source of truth.
