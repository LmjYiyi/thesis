# DOCX Sync Workflow

## Scope

This skill is intentionally narrow. Use it for section-level thesis updates where the Markdown already contains the final wording and the master document is a `.docx` file with stable heading styles.

Supported in v3:

- heading-bounded section replacement
- chapter-level sync after merging multiple Markdown section files into one temporary chapter file
- front-matter anchors that use a non-numbered title style, such as `摘要` and `ABSTRACT`
- Markdown headings whose visible numbering should be stripped because the target Word heading style already numbers them
- body paragraphs
- ordered and unordered lists
- standard Word tables produced by `pandoc`
- inline and display math rendered by `pandoc`
- display equations that include `\tag{...}` in Markdown
- Markdown image references that can be resolved to local files inside the thesis workspace
- Markdown image references whose local file paths are broken, as long as a source `--image-source-docx` contains the needed figure and the Markdown still carries a figure number such as `图5-1`
- Explicit `docx-figure://图5-1` image placeholders that should be filled from a source DOCX instead of a local file
- optional downgrade of unresolved Markdown images into visible placeholder text
- footnotes and endnotes generated by `pandoc`
- external hyperlinks generated by `pandoc`
- internal anchors, bookmarks, and in-document jump links generated from Markdown IDs or `#anchor` links
- table of contents, figure index, and table index refresh compatibility after Word field update
- preflight checking for off-policy image paths and risky inline math before sync

Still not supported:

- legacy `.doc`
- tracked changes
- Word comments
- embedded OLE objects inside the Markdown fragment
- Markdown fragments whose image paths cannot be resolved to real local files
- equation-number layout that must exactly mimic Word right-aligned professional equation templates

## Recommended Flow

1. Work on the chapter in Markdown until the content is ready.
2. If the chapter is split into multiple Markdown files, merge them first with `build_markdown_chapter.py`. Put the merged file near the source files so relative image paths such as `figures/...` still resolve.
3. Run `preflight_markdown_sync.py` before sync. For this thesis, the recommended profile is `final_output_doc`, which enforces canonical local figures under `figures/final_output_doc/` and flags risky inline formulas:

```bash
python skills/thesis-docx-sync/scripts/preflight_markdown_sync.py ^
  --markdown writing/_chapter4_sync_tmp.md ^
  --profile final_output_doc ^
  --fail-on-warning
```

4. If the Markdown's image file paths are unreliable, point `--image-source-docx` at a thesis DOCX that already contains the final figures. Use `docx_figure_catalog.py` first if you need to see which figure numbers the source DOCX exposes.
5. Run `dump_docx_outline.py` on the target master file and copy the exact heading text you want to replace.
6. If the master file is an older thesis version, confirm the target block by surrounding headings and body position instead of assuming the chapter numbering still matches the Markdown numbering.
7. Run `sync_markdown_to_docx.py` with that heading as the anchor.
8. If the output file is open in Word, close it or write to a new output filename first.
9. Open the output `.docx` in Word and refresh table of contents, figure index, and table index if needed.
10. Do a fast visual check on formulas, tables, paragraph spacing, chapter breaks, and figure sizing.
11. If a formula used `\tag{...}`, confirm the number is still visually acceptable after conversion.
12. If the chapter contains footnotes or internal jump links, click a few representative items in Word to confirm they still resolve correctly.

## Why This Works

The sync script does not regenerate the whole thesis document. It only replaces the XML block between one heading and the next sibling heading of the same or higher level. This keeps the rest of the master file unchanged, including front matter, references, and other chapters.

`pandoc` is used only to render Markdown into a Word fragment that already follows the style universe of the thesis master document through `--reference-doc`.

Before rendering, the script rewrites display equations containing `\tag{...}` into a `pandoc`-compatible form and stages Markdown images into a temporary asset folder. If a local image path cannot be resolved, the script can optionally fall back to a source DOCX and pull the figure by matching the Markdown's figure number against nearby figure captions in that DOCX. During DOCX merge, it also carries over document relationships, bookmark IDs, and note parts so images, hyperlinks, footnotes, endnotes, and internal jump targets survive the section replacement.

When the target anchor uses a non-numbered title style instead of a numbered heading style, the script also aligns the first inserted heading paragraph to that target style so abstract-like sections keep the template's original appearance.

The preflight step exists because Word is much less tolerant than Markdown for two patterns: off-policy image paths and dense inline formulas. In this thesis project, final images should point at `writing/figures/final_output_doc/` and helper artifacts must never be written back into that canonical directory. For body text, simple inline fractions should usually be rewritten in slash notation, while numbered formulas should remain display equations.

## Failure Modes

- If the heading cannot be found, dump the outline first and use the exact text from the target `.docx`.
- If the heading is ambiguous, rerun with `--level`.
- If the replacement lands in the wrong chapter, the likely cause is a stale master `.docx` whose chapter order no longer matches the Markdown plan. Re-check the actual Word outline and surrounding headings before rerunning.
- If the chapter is split across several Markdown files, merge them first instead of trying to sync each subsection into a chapter-level anchor manually.
- If the script reports unsupported features, move comments or OLE objects back to the master document manually and sync the rest.
- If the script reports unresolved image paths, fix the Markdown path or move the referenced image into the workspace first.
- If preflight reports an image outside the canonical figure directory, fix the Markdown to point at the approved figure library instead of creating helper copies in that directory.
- If preflight reports inline `\frac` or oversized inline math, flatten the body-text expression to slash notation or move it to display math before sync.
- If the source Markdown uses broken image paths but still includes alt text such as `图5-1 ...`, rerun with `--image-source-docx`.
- If the source DOCX fallback still cannot find a figure, run `docx_figure_catalog.py source.docx` and compare the actual figure numbers it discovered against the Markdown text.
- If the source images are genuinely unavailable but the text sync still needs to continue, rerun with `--missing-images placeholder`.
- If the script reports the output DOCX is locked or open, close the file in Word or change `--output`.
- If the table of contents or figure index looks stale after sync, update fields in Word instead of editing those index pages manually.
- If the master file is still `.doc`, convert it to `.docx` first and treat the converted file as the new single source of truth.
